New updates for CodePush and React Native Module for CodePush

27 Mar 2017 - byIan Geoghegan

Over the last thirty days, we’ve released several significant updates to both CodePush and the React Native Module for CodePush that we think you’ll like. CodePush boasts several new features, including the abililty to specify Node parameters to the release-react command, the ability to specify bundle and sourcemap destination, deployment promotion based on labels, and a release build type for Apache Cordova builds. The React Native Module for CodePush now includes support for React Native Windows .NET 4.6. Both products also have seen numerous bug fixes. Check out CodePush and the Module release notes for details and the new and improved documentation for the React Native Module for CodePush.

Big Plans

In addition to the features and fixes, we’re also announcing our plan to migrate CodePush to Visual Studio Mobile Center. This will let existing users continue to use the same product they know and love, while adding the tooling and services provided by Mobile Center. We could go on about the product’s Build, Test, Distribute, Crash, and Analytics services, but why bother when we made videos?
